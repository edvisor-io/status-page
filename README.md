# Edvisor Status Page

Public status page for [status.edvisor.io](https://status.edvisor.io) - displays real-time service health and uptime metrics.

## Features

- Real-time service status monitoring
- 90-day uptime history visualization
- Incident timeline with updates
- Auto-refresh every 60 seconds
- Responsive design

## Tech Stack

- **Framework:** React 18 + TypeScript
- **Build Tool:** Vite 5
- **Styling:** TailwindCSS 4
- **Icons:** Lucide React
- **Hosting:** S3 + CloudFront

## Development

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

## Data Sources

The status page reads from static JSON files:

- `status.json` - Service status and uptime data
- `incidents.json` - Active and recent incidents

These files are generated by the Developer Portal's StatusSync service.

## Deployment

Deployments are handled automatically via CircleCI:

1. Push to `main` branch triggers build
2. Static files are synced to S3 bucket
3. CloudFront cache is invalidated

### Environment Variables (CircleCI)

| Variable | Description |
|----------|-------------|
| `AWS_ACCESS_KEY_ID` | AWS credentials |
| `AWS_SECRET_ACCESS_KEY` | AWS credentials |
| `AWS_DEFAULT_REGION` | `us-west-2` |
| `CLOUDFRONT_DISTRIBUTION_ID` | CloudFront distribution ID |

## Architecture

```
Developer Portal (internal) --> StatusSync cron --> S3 bucket
                                                      |
Status Page (this repo) --> Build --> S3 <-- CloudFront --> status.edvisor.io
```
